# v7B.1.1: Cloud SQL Editor Canary Workflow

**For:** Operators without terminal access (cloud-only)  
**Method:** Supabase Web SQL Editor (authenticated via dashboard OAuth)  
**Scope:** Single canary INSERT + SELECT readback + cleanup

---

## Why This Works

The Supabase Web SQL Editor runs as the `postgres` role, authenticated through your Supabase dashboard login. You do NOT need to paste the `service_role` key into the editor — your dashboard session is the authentication.

---

## Step 1: Canary INSERT

In your Supabase SQL Editor, create a new query and run:

```sql
-- v7B.1.1: Canary INSERT into public.memories
-- Scope: single test row, non-sensitive metadata, zero vector
INSERT INTO public.memories (
    id,
    content,
    metadata,
    embedding
) VALUES (
    gen_random_uuid(),                          -- random UUID
    'v7B.1.1-canary | corrected-endpoint | ' || now()::text,
                                                -- identifiable content
    '{                                          -- JSONB metadata
        "source": "v7b1.1-canary",
        "version": "7.1.1",
        "seal": "sfa-barbell-dashboard-v7b1.1-canary",
        "notExecutionAuthority": true,
        "containsTradeOrders": false,
        "isGovernedState": false,
        "operator": "cloud-sql-editor"
    }'::jsonb,
    array_fill(0, ARRAY[768])::vector          -- 768d zero vector
)
RETURNING id, content, metadata, created_at;
```

**Expected result:** One row returned with `id`, `content`, `metadata`, and `created_at`.

**Capture:** Screenshot the result. Note the returned `id` UUID.

---

## Step 2: Readback Verification

In a new query, run:

```sql
-- v7B.1.1: Readback verification
-- Replace ? with the UUID returned from the INSERT above
SELECT 
    id,
    content,
    metadata,
    created_at
FROM public.memories
WHERE content LIKE 'v7B.1.1-canary%'
ORDER BY created_at DESC
LIMIT 5;
```

**Expected result:** Your canary row appears in the results.

**Capture:** Screenshot showing the row.

---

## Step 3: (Optional) Cleanup

```sql
-- v7B.1.1: Remove canary test rows
-- Only deletes rows created by this canary
DELETE FROM public.memories
WHERE content LIKE 'v7B.1.1-canary%';
```

This is safe — the `WHERE` clause only matches your canary rows.

# v7B.1.3: Open Brain Memory Proposal Queue

**Phase:** v7B.1.3 — Local Memory Proposal Generation, Validation, Review  
**Status:** Complete — 40/40 tests passed  
**Scope:** Local artifacts only. No database writes. No live INSERT/UPDATE/DELETE.

---

## Purpose

The proposal queue is the **human-review gate** between manual canary writes (v7B.1.1) and any future controlled automation (v7B.2+). It ensures:

1. Every proposed memory is validated before review
2. Every proposed memory is safety-classified before review
3. Every proposed memory requires explicit human approval before promotion
4. No proposal can auto-write to Open Brain

---

## Architecture

```
Proposal Generator
  → Proposal Validator (fail-closed, 13 checks)
    → Safety Classifier (credential/trade/strategy scan)
      → Human Review Ledger (proposed → approved/rejected/needs_revision)
        → Promotion Packet Generator (SQL/REST-ready, NOT executed)
          → Operator manual execution (SQL Editor)
```

## Components

### 1. Proposal Schema (`src/bridge/v7b/proposalSchema.ts`)

| Field | Required | Description |
|-------|----------|-------------|
| `proposalId` | UUID | Unique identifier |
| `content` | string | Memory content (max 10,000 chars) |
| `metadata.source` | string | Source attribution |
| `metadata.confidence` | 0.1–1.0 | Confidence score |
| `metadata.governance` | object | Six boolean flags (all must be safe) |
| `metadata.governance.isGovernedState` | **false** | Must be false |
| `metadata.governance.containsTradeOrders` | **false** | Must be false |
| `metadata.governance.notExecutionAuthority` | **true** | Must be true |
| `metadata.governance.containsCredentials` | **false** | Must be false |
| `metadata.governance.containsWalletReferences` | **false** | Must be false |
| `metadata.governance.isStrategyInstruction` | **false** | Must be false |

### 2. Proposal Validator (`src/bridge/v7b/proposalValidator.ts`)

**13 fail-closed checks.** Any single failure blocks the proposal.

| # | Check | Blocks If |
|---|-------|-----------|
| 1 | content_present | Empty or missing |
| 2 | content_length | > 10,000 characters |
| 3 | source_present | Missing or empty |
| 4 | confidence_range | < 0.1 or > 1.0 |
| 5 | governance_present | Missing entirely |
| 6 | not_governed_state | `isGovernedState !== false` |
| 7 | no_trade_orders | `containsTradeOrders !== false` |
| 8 | not_execution_authority | `notExecutionAuthority !== true` |
| 9 | no_credentials | `containsCredentials !== false` |
| 10 | no_wallet_refs | `containsWalletReferences !== false` |
| 11 | no_strategy | `isStrategyInstruction !== false` |
| 12 | proposal_id | Missing |
| 13 | version_valid | Not v7b-prefixed |

### 3. Safety Classifier (`src/bridge/v7b/safetyClassifier.ts`)

Content scanning for dangerous patterns:

| Pattern Type | Examples Detected |
|-------------|-------------------|
| `CREDENTIAL_LEAK` | `sbp_*`, `sk-*`, `pk-*`, JWT tokens, `private_key`, `api_key=...` |
| `TRADE_ORDERS` | `buy 100 shares`, `go long`, `market order`, `leverage 10x` |
| `WALLET_REFERENCE` | `0x742d35Cc6634...` (Ethereum addresses) |
| `STRATEGY_OVERRIDE` | `override risk threshold`, `change stop loss`, `ignore risk` |
| `GOVERNED_STATE` | `isGovernedState: true` in metadata |
| `CLAIMS_EXECUTION_AUTHORITY` | `notExecutionAuthority: false` in metadata |

### 4. Review Ledger (`src/bridge/v7b/reviewLedger.ts`)

| Status | Meaning | Who Can Set |
|--------|---------|-------------|
| `proposed` | Awaiting review | System (default) |
| `approved_for_manual_write` | Ready for promotion packet | Human operator only |
| `rejected` | Will not be promoted | Human operator only |
| `needs_revision` | Requires changes, then re-review | Human operator only |

**Promotion readiness requires ALL THREE gates:**
1. ✅ Validation passed
2. ✅ Safety classified as safe
3. ✅ Review status = `approved_for_manual_write`

### 5. Promotion Packet (`src/bridge/v7b/promotionPacket.ts`)

Generates a **ready-to-execute** packet that the operator runs manually:

```sql
-- Example promotion packet SQL
INSERT INTO public.memories (id, content, metadata, embedding)
VALUES ('<proposal-id>', '<content>', '<metadata>'::jsonb, array_fill(0, ARRAY[768])::vector)
RETURNING id, content, metadata, created_at;
```

**The packet is NEVER executed automatically.** The operator must:
1. Review the SQL
2. Verify the governance attestation
3. Copy to Supabase SQL Editor
4. Execute manually
5. Verify with SELECT
